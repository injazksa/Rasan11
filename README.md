# منصة رَسَن (Rasan) - نظام إدارة الخيول الشامل 📱

> **تنبيه لمستخدمي الهاتف:** تم إضافة مجلد `live_preview` لتشغيل الواجهة مباشرة دون الحاجة لأوامر برمجية. راجع قسم "نسخة الهاتف" أدناه.

## نظرة عامة

منصة رَسَن هي نظام متكامل لإدارة قطاع الفروسية والخيول، يجمع بين السلطة التنظيمية (الاتحادات) والتجارة الحصرية (المتجر) والسوق المفتوح (الملاك) والخدمات الطبية والمزادات الحية.

### الميزات الرئيسية

#### 1. نظام المصادقة والصلاحيات (RBAC)
- **5 أدوار رئيسية:**
  - **مدير النظام (Super Admin):** التحكم الكامل بالمنصة والمستخدمين والمالية
  - **الاتحادات (Federations):** إدارة الخيول والملاك والسباقات في دولتهم
  - **الملاك (Owners):** إدارة الخيول والإسطبل والبيع والشراء
  - **الأطباء البيطريون (Doctors):** إدارة الوصفات الطبية والسجلات الصحية
  - **البائعون (Vendors):** عرض المنتجات والخدمات

#### 2. لوحات التحكم المتخصصة
- **لوحة المدير:** إحصائيات شاملة، إدارة المستخدمين، المراقبة المالية
- **لوحة الاتحاد:** إدارة الخيول والملاك، السباقات، العقوبات والغرامات
- **لوحة الطبيب:** إصدار الوصفات، تتبع العلاجات، السجلات الصحية
- **إسطبل المالك:** إدارة الخيول، الوثائق، الجواز الرقمي
- **متجر البائع:** عرض المنتجات، إدارة الطلبات

#### 3. نظام المزادات الحية
- مزايدة فورية للخيول مع تحديث لحظي
- تأمين الجدية لضمان الجدية في المزايدة
- نسب عمولة متغيرة للمدير

#### 4. السوق والتجارة
- متجر مركزي حصري للأدوية والمعدات
- نظام الإعلانات المميزة
- نظام البحث والفلترة المتقدم

#### 5. الجواز الرقمي للخيول
- صفحة عامة فخمة مع QR Code
- سجل كامل للحصان (النسب، الصحة، الإنجازات)
- ختم رقمي موثق من الاتحاد

#### 6. النظام المالي
- بوابة دفع آمنة (Stripe + Apple Pay)
- تقسيم العمولة الآلي
- نظام Escrow للحجز المؤقت
- سجل كامل للمعاملات

#### 7. إدارة الوصفات الطبية
- إصدار وصفات رقمية من الأطباء
- تفعيل شراء الأدوية المقيدة
- تتبع العلاجات والسجلات الصحية

#### 8. نظام الإشعارات الذكية
- إشعارات فورية للمستخدمين
- WebSocket للتحديثات المباشرة
- تصنيفات حسب نوع الحساب

## البدء السريع

### المتطلبات
- Node.js 16+ و npm/pnpm
- MySQL 8.0+ أو PostgreSQL
- Redis (اختياري للجلسات)

### التثبيت

```bash
# 1. استنساخ المشروع
git clone <repo-url>
cd rasan

# 2. تثبيت المتطلبات
cd frontend && npm install
cd ../backend && npm install

# 3. إعداد متغيرات البيئة
cp frontend/.env.example frontend/.env
cp backend/.env.example backend/.env

# 4. إنشاء قاعدة البيانات والهجرات
npm run db:migrate

# 5. تشغيل الخادم الخلفي
cd backend && npm run dev

# 6. تشغيل الواجهة الأمامية (في نافذة أخرى)
cd frontend && npm run dev
```

## نظام الصلاحيات (RBAC)

### الأدوار والصلاحيات

```
Super Admin
├── عرض جميع المستخدمين
├── إدارة الاتحادات
├── المراقبة المالية الكاملة
├── الدخول بصفة مستخدم (Impersonation)
└── إدارة السجلات والتدقيق

Federation
├── إدارة الخيول في دولتهم
├── إدارة الملاك
├── إصدار الغرامات والعقوبات
├── نشر أخبار السباقات
└── الموافقة على الإعلانات

Owner
├── إدارة الخيول الخاصة
├── عرض الخيول للبيع
├── المشاركة في المزادات
├── شراء المنتجات
└── عرض الجواز الرقمي

Doctor
├── إصدار الوصفات الطبية
├── تتبع العلاجات
├── عرض السجلات الصحية
└── إدارة المرضى

Vendor
├── عرض المنتجات
├── إدارة الطلبات
└── تتبع المبيعات
```

## واجهات برمجية (APIs)

### المصادقة
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/register` - إنشاء حساب
- `POST /api/auth/logout` - تسجيل الخروج
- `GET /api/auth/me` - بيانات المستخدم الحالي

### الخيول
- `GET /api/horses` - قائمة الخيول
- `POST /api/horses` - إضافة خيل جديد
- `GET /api/horses/:id` - تفاصيل الخيل
- `PUT /api/horses/:id` - تحديث الخيل
- `DELETE /api/horses/:id` - حذف الخيل

### المزادات
- `GET /api/auctions` - قائمة المزادات
- `POST /api/auctions` - إنشاء مزاد
- `POST /api/auctions/:id/bid` - وضع مزايدة
- `GET /api/auctions/:id` - تفاصيل المزاد

### الوصفات الطبية
- `GET /api/prescriptions` - قائمة الوصفات
- `POST /api/prescriptions` - إصدار وصفة
- `GET /api/prescriptions/:id` - تفاصيل الوصفة

### المتجر
- `GET /api/marketplace` - قائمة المنتجات
- `POST /api/marketplace/items` - إضافة منتج
- `POST /api/marketplace/orders` - إنشاء طلب

### المعاملات المالية
- `GET /api/transactions` - سجل المعاملات
- `POST /api/transactions/payment` - معالجة الدفع
- `GET /api/transactions/:id` - تفاصيل المعاملة

## نسخة الهاتف (Mobile Ready) 📱

لقد تم تجهيز هذا المشروع ليعمل بأكثر من طريقة من الهاتف:

### 1. التشغيل المباشر (Live Preview)
إذا كنت تريد تشغيل الموقع فوراً:
- ارفع محتويات مجلد `live_preview` إلى أي استضافة ملفات عادية.
- افتح رابط الموقع وسيعمل فوراً.

### 2. النسخة الكاملة (Node.js)
- ارفع المشروع كاملاً.
- ادخل لمجلد `backend` وشغل الأمر: `npm start`.
- جميع المكتبات (`node_modules`) مثبتة مسبقاً.

### محتويات المجلد المضافة:
- `live_preview/`: ملفات HTML/JS/CSS جاهزة للرفع المباشر.
- `backend/node_modules/`: المكتبات مثبتة وجاهزة.
- `frontend/node_modules/`: المكتبات مثبتة وجاهزة.
