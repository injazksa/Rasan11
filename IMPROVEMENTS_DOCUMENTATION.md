# توثيق التحسينات الشاملة لمنظومة رَسَن - Rasan11

## 📋 ملخص التحسينات

تم إجراء تحسينات شاملة على لوحة تحكم مدير النظام والواجهة الأمامية لتفعيل جميع الأزرار والأوامر الحقيقية وإضافة ميزات متقدمة.

---

## 🎯 الميزات المضافة والمحسّنة

### 1. **لوحة تحكم المدير المحسّنة (AdminMasterDashboard)**

#### الميزات الجديدة:

##### أ) **إدارة المستخدمين الشاملة**
- ✅ **تعديل بيانات المستخدم** - يمكن للمدير تعديل جميع بيانات المستخدم (الاسم، البريد، الهاتف، الدولة، المدينة)
- ✅ **إرسال إيميلات** - إرسال رسائل مباشرة للمستخدمين مع خيارات متعددة (إشعار، تحذير، تحديث مهم)
- ✅ **طلب مستندات** - طلب وثائق محددة من المستخدمين مع ملاحظات
- ✅ **إرسال رابط إعادة تعيين كلمة المرور** - إرسال رابط آمن للمستخدم لإعادة تعيين كلمته
- ✅ **تغيير الصلاحيات** - تعديل دور المستخدم (مدير، اتحاد، مالك، طبيب، بائع، مشرف)
- ✅ **حظر/فك حظر** - حظر أو فك حظر المستخدمين مع تسجيل السبب

##### ب) **نظام الإشعارات المتقدم**
- ✅ **جرس الإشعارات** - عرض عدد الإشعارات غير المقروءة
- ✅ **قائمة منسدلة للإشعارات** - عرض آخر 50 إشعار مع حالة القراءة
- ✅ **تحديث تلقائي** - تحديث الإشعارات كل 30 ثانية
- ✅ **تحديث حالة الإشعار** - تحديث الإشعار كمقروء عند الضغط عليه

##### ج) **إدارة طلبات الموافقة**
- ✅ **قبول/رفض المستخدمين** - الموافقة على المستخدمين الجدد أو رفضهم
- ✅ **طلب مستندات إضافية** - طلب وثائق معينة قبل الموافقة
- ✅ **إرسال رسائل** - التواصل مع المستخدمين المنتظرين

##### د) **نظام الصلاحيات المرتب**
- ✅ **عرض الصلاحيات** - عرض جميع الأدوار والصلاحيات المرتبطة بها
- ✅ **تسلسل هرمي واضح** - من الأقوى (super_admin) إلى الأضعف (doctor)

##### هـ) **البحث والتصفية**
- ✅ **بحث متقدم** - البحث بالاسم أو البريد أو رقم الـ ID
- ✅ **تصفية بالدور** - تصفية المستخدمين حسب الدور
- ✅ **تصفية بالحالة** - تصفية حسب نشط/غير نشط/محظور/معلق
- ✅ **تصفية بالدولة** - تصفية حسب الدولة

##### و) **الإجراءات الجماعية**
- ✅ **حظر جماعي** - حظر عدة مستخدمين في نفس الوقت
- ✅ **فك حظر جماعي** - فك حظر عدة مستخدمين في نفس الوقت

### 2. **الشريط العلوي المحسّن (Navigation)**

#### التحسينات:

- ✅ **قائمة منسدلة ذكية** - عرض أول 3 عناصر مباشرة والباقي في قائمة منسدلة
- ✅ **إغلاق تلقائي** - إغلاق القائمة المنسدلة عند الضغط على عنصر أو خارجها
- ✅ **تصميم محسّن** - تصميم أفضل وأكثر احترافية
- ✅ **استجابة للجوال** - تصميم متجاوب يعمل على جميع الأجهزة
- ✅ **مؤشرات بصرية** - تمييز الصفحة الحالية بشكل واضح

### 3. **مسارات API الجديدة (Backend)**

#### المسارات المضافة:

```
PUT  /api/admin/user/:userId
     - تحديث بيانات المستخدم

POST /api/admin/user/:userId/send-email
     - إرسال إيميل للمستخدم

POST /api/admin/user/:userId/request-document
     - طلب مستند من المستخدم

POST /api/admin/user/:userId/send-password-reset
     - إرسال رابط إعادة تعيين كلمة المرور

POST /api/admin/user/:userId/permissions
     - تغيير صلاحيات المستخدم

GET  /api/admin/approvals
     - جلب طلبات الموافقة

GET  /api/admin/auctions
     - جلب المزادات (للمدير)

GET  /api/admin/products
     - جلب المنتجات (للمدير)

POST /api/admin/products
     - إنشاء منتج جديد

PUT  /api/admin/products/:productId
     - تحديث المنتج

DELETE /api/admin/products/:productId
     - حذف المنتج

GET  /api/notifications
     - جلب الإشعارات

POST /api/notifications/:notificationId/read
     - تحديث الإشعار كمقروء
```

## 📧 نظام الإيميلات

### الإيميلات المدعومة:

1. **إيميل الموافقة على المستخدم**
   - يُرسل عند قبول طلب المستخدم
   - يحتوي على رسالة ترحيب ملكية

2. **إيميل رفض المستخدم**
   - يُرسل عند رفض طلب المستخدم
   - يحتوي على سبب الرفض

3. **إيميل تحديث كلمة المرور**
   - يُرسل عند تحديث كلمة المرور من قبل المدير
   - يحتوي على تنبيه أمان

4. **إيميل رسالة مخصصة**
   - يُرسل رسالة مخصصة من المدير
   - يدعم أنواع مختلفة (إشعار، تحذير، تحديث)

5. **إيميل طلب مستند**
   - يُرسل طلب مستند محدد
   - يحتوي على ملاحظات إضافية

6. **إيميل إعادة تعيين كلمة المرور**
   - يُرسل رابط آمن لإعادة تعيين كلمة المرور
   - الرابط صالح لمدة 24 ساعة

7. **إيميل تحديث الصلاحيات**
   - يُرسل عند تغيير دور المستخدم
   - يحتوي على الدور الجديد

## 🛠️ التثبيت والتشغيل

### متطلبات البيئة:

```bash
Node.js >= 18.0.0
npm أو yarn أو pnpm
قاعدة بيانات PostgreSQL أو MySQL
```

### خطوات التثبيت:

```bash
# 1. استنساخ المشروع
git clone https://github.com/injazksa/Rasan11.git
cd Rasan11

# 2. تثبيت الحزم
npm install

# 3. إعداد متغيرات البيئة
cp .env.example .env
# عدّل ملف .env بالقيم الصحيحة

# 4. تشغيل الخادم
npm start

# 5. تشغيل الواجهة الأمامية
cd frontend
npm install
npm run dev
```

### متغيرات البيئة المهمة:

```env
# قاعدة البيانات
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=password
DB_NAME=rasan

# البريد الإلكتروني
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASS=your-app-password

# JWT
JWT_SECRET=your-secret-key

# Frontend URL
FRONTEND_URL=http://localhost:5173
```
